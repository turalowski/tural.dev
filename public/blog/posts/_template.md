Open with the problem or claim in the first paragraph. Search engines and readers both weight the start heavily — don't bury the point under context.

## Section heading that matches search intent

Use `##` / `###` in the body only. The page template already renders `title` as the single `<h1>`, so do not put another `#` heading in the markdown.

- Keep paragraphs short
- Prefer concrete examples over abstract claims
- One idea per section

### Sub-section when you need to go deeper

```ts
// Short, focused code samples beat walls of code

```

> Use blockquotes for the one sentence a skimmer should remember.

Close with a clear takeaway or decision checklist — that is what people quote, share, and search for later.
